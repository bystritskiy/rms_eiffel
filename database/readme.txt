Пока что бд представляет из себя одну таблицу с кучей столбцов (все 22 поля формы + 1 id). Такая конструкция подразумевает, 
что при первом запуске заполнения формы пользователь вводит данные об отчёте и себе, сабмитит и тем самым на сервере происходит 
создание новой строки в таблице, а пользователю возвращается его id. Id сохраняется в cookie, которые присылаются серверу от пользователя 
каждый раз с каждым сабмитом. Тем самым мы по id можем заполнить и другие столбцы сабмитами из других отделов. 

Возможно одна большая таблица будет неудобна, но всё же пока так живём. Будет удобнее её возвращать на admin page данные.
